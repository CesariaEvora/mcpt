# Contributing

## Working Model
- Start from a written plan with explicit success criteria.
- Keep PRs focused on one outcome.
- Include validation evidence and risk notes in each PR.

## Definition of Ready
- Problem and target user are clear.
- Scope and non-goals are explicit.
- Dependencies and blockers identified.

## Definition of Done
- Quality gates pass.
- Failure modes tested.
- Docs/steering updated for changed behavior.

## PR Checklist
- [ ] Scope matches approved plan
- [ ] Tests/verification completed
- [ ] Security/compliance impact reviewed
- [ ] Rollback/mitigation documented
