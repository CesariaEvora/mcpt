# OpenClaw Steering Charter

## Mission
Ship customer outcomes quickly without sacrificing reliability, trust, or operational safety.

## Priorities
1. Customer value and measurable outcomes
2. Reliability of core workflows
3. Security/compliance by default
4. Maintainability and repeatability

## Decision Rules
- Prefer simple systems with clear ownership.
- Push complexity to reusable templates and runbooks.
- Do not promote to production without explicit acceptance evidence.

## Standard Artifacts Per Project
- Master plan
- Decision log
- Risk register
- Production readiness checklist
- Release runbook
