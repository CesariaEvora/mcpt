# GitHub Steering Pack

## Required Markdown Files
- `.github/pull_request_template.md`
- `.github/ISSUE_TEMPLATE/bug_report.md`
- `.github/ISSUE_TEMPLATE/feature_request.md`

## Operating Rules
- Every PR must include risk and validation evidence.
- Every bug report must include reproduction and impact.
- Every feature request must include measurable success criteria.

## Recommendation
Keep these templates identical across customer repos to reduce onboarding overhead and improve review consistency.
