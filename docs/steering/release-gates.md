# Release Gates

## Gate A: Plan Complete
- [ ] Scope and acceptance criteria approved
- [ ] Dependencies and blockers documented
- [ ] Risk and rollback strategy present

## Gate B: Build Complete
- [ ] Critical path implemented
- [ ] Validation and failure handling in place
- [ ] Docs updated to match implementation

## Gate C: Production Ready
- [ ] Domain and email auth ready (SPF/DKIM/DMARC if applicable)
- [ ] Secrets configured securely
- [ ] Smoke tests pass in production-like environment

## Gate D: Post-Release
- [ ] Monitoring checks complete
- [ ] Error rates within acceptable range
- [ ] Follow-up issues logged and prioritized
